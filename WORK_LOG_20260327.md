# 작업 일지 — 2026-03-27

## 1. 업무일지 전면 개편 (salesMgmt.html)

### 목록 페이지
- 일 단위 게시글 + 연도/월 탭 (Firestore `salesDiaries` 연동)
- 일별 카드: 업체 칩(기존/신규) + 매출 합계 + 임시저장 뱃지
- 통계 카드: 이번 달 작성/방문/매출/신규영업

### 상세/작성 통합 페이지
- 6개 섹션: 방문 업체 / 매출 합계 / 경쟁사 동향 / 신규영업 / 비고 / 익일 계획
- 업체 카드: 기존/신규 토글 + 소개경로(신규만) + 주소 자동 로드
- 검사목적별 매출: 3열 가로 그리드, 부가세 포함 입력, 접수취소 제외
- 합계 블록: 목적별 합산 + 공급가액/세액 구분 표시
- 파일 첨부: 경쟁사 동향 + 특이사항 각각
- 비교견적 토글: 1:1:1:1 균등배분 (항목/경쟁사/우리/차액), 자동 콤마, 제안사항 박스

### 익일 계획 3탭
- **기존 거래처 방문**: 1:1:1:1 균등배분 + 원(부가세포함) + 예상매출 합계
- **신규 영업**: 3모드 (업체명 검색 / 지역 탐색 / 신규업체 알림)
  - 공통 업종 드롭다운 + [조회] 버튼 → fssAllData 캐시 (업체조회와 동일 방식)
  - 업체명 검색: fssAllData 로컬 필터 (즉시 응답)
  - 지역 탐색: regionData + fssAllData 로컬 필터, 시도→시군구 드릴다운, 업체 수 표시
  - 신규업체 알림: prms_dt(인허가일) 기준 20일 이내 (경과일 색상: 7일=초록, 14일=주황, 20일=빨강)
- **메모**: 자유 텍스트

### 상단 탭 복원
- 기존 6개 탭(업무일지/미수금/거래명세표/계산서/세금계산서/차량일지) 복원
- switchDailyTab 함수 실제 동작 복구
- 계산서발행승인/세금계산서미발행 Firestore 연동 유지

### Firestore 연동
- 컬렉션: `salesDiaries`, 문서ID: `YYYYMMDD`
- `date` 필드 기반 월별 범위 쿼리 (인덱스 불필요)
- 임시저장/저장완료 상태 관리
- 업체별 목적매출 + 비교견적 + 제안사항 저장

### 식약처 데이터 호출 방식 통일
- 기존: 매 검색마다 Firestore 쿼리 → 변경: fssAllData 캐시 + 로컬 필터
- `fss_businesses` 필드명 교정: `bssh_nm`, `locp_addr`, `induty_nm`, `addr_sido`, `prms_dt`
- `loadTmrFssData()`: 업체조회 `loadFssData()`와 동일 방식
