# 작업 일지 — 2026-03-23

## 1. 접수등록 (sampleReceiptV2.html) UI 전면 개편

### 레이아웃 변경
- 상단에 '접수 관리 > 접수 등록' breadcrumb 추가
- API 연결 상태 표시를 breadcrumb 오른쪽으로 이동
- 왼쪽 폼 너비 500px → 620px 확장 (3열 배치 공간 확보)
- 좌우 패널 드래그 리사이저 추가 (400~900px, localStorage 저장)

### 왼쪽 폼 3열 배치
- 1행: 제품명 / 보고번호 / 검체유형
- 2행: 제조일자 / 소비기한(일수) / 소비기한
- 3행: 롯트번호 / 포장단위 / 검체수
- 4행: 시료접수 / 접수형태 / 운반상태
- 5행: 공지여부 / 지부 / 라벨

### 인라인 라벨 UI 통일
- **포장단위**: 숫자 입력 안 `g` 라벨 → 클릭 시 가로 펼침 (g/kg/mL/L/ea, 색상 구분)
- **소비기한(일수)**: `일` 라벨 → 클릭 시 일↔년 토글 (년=주황)
- **시료접수**: 라벨 클릭 → 내방/택배/바이오/일반 가로 펼침
- **접수형태**: 라벨 클릭 → 일반/홈페이지 가로 펼침
- **운반상태**: 라벨 클릭 → 냉장/냉동/실온/상온 가로 펼침 (3열이라 좌측 펼침)
- **공지여부**: 라벨 클릭 → 예/아니요/메모 가로 펼침
- **긴급여부**: 상단 헤더바 상태 오른쪽으로 이동, 라벨 클릭 → 아래 펼침
- **제품명/보고번호**: 버튼 → 인라인 라벨 (📋/🔍)

### 공지여부 세부 동작
- "예" → 접수공지 모달 호출
- "아니요" → 접수공지 박스 숨김
- "메모" → 전체 너비 메모 텍스트박스 (placeholder: "메모에 기록된 내용은 공지 대상이 아닙니다")

### 지부/라벨 인라인 UI
- **지부**: select → 인라인 검색 (지부명/사업자번호, 드롭다운 선택)
- **라벨**: select+버튼 → 인라인 피커 (클릭→체크리스트, 입력 안 칩 표시, ✕ 제거)

### 삭제/이동
- 팀별 메모 4개 박스 삭제 (hidden으로 데이터 호환)
- 하단 계산서/결제입금 버튼+패널 삭제 (hidden으로 데이터 호환)
- 접수번호 할당 경고 박스 삭제
- 첨부파일 원래 위치(시료위치 위) 유지

### 성적서 발송
- 우편 발송 패널에 '🏢 지부 주소' 체크박스 추가 (지부 검색/선택)

### OCR 연동 보장
- hidden input id 유지, syncInlineUI() 함수로 OCR 설정 후 UI 동기화
- selectUnit() 호출로 포장단위 라벨 갱신
