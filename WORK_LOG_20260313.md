- **문제**: I2859/I2860 API에서 변경이력이 거의 수집 안 됨 (3건/2건)
- **원인**: 식약처 API가 `1~1` 범위로 요청 시 `total_count`를 부정확하게 반환 (6건)하지만, `1~1000` 범위로 요청하면 정확한 값(2,778건) 반환
- **수정 내용** (서버 collector.py 직접 수정):
  1. 변경이력 API 총 건수 조회: `1~1` → `1~1000`으로 변경 (정확한 total_count 확보)
  2. `collect_api_auto`도 동일 수정 (cron 자동 수집 시에도 적용)
  3. `insert_change` 중복 방지: 자동 ID → `인허가번호_변경일자_변경사유` 고유 ID 방식
  4. 첫 배치 재활용: 총 건수 확인 시 받은 데이터를 재사용하여 API 호출 절약
- **결과**: I2859 3건→2,778건, I2860 2건→1,000건, 변경이력 합계 20건→3,035건
