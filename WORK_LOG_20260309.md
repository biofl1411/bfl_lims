- **원인**: nginx `client_max_body_size` 기본 1MB (PDF base64 >> 1MB), `/fss/` 타임아웃 미설정

## 세션 3: 접수등록 참고용 카테고리 라벨 칩 기능

### 카테고리 라벨 칩 추가
- **목적**: 참고용 검사목적 선택 시 300+ 항목을 하나씩 검색하는 번거로움 해소
- **구현**: 시료 라벨 아래에 소분류별 라벨 칩(P1~P9) 표시
  - P1:이화학(73), P2:영양/기능(80), P3:미생물(82), P4:중금속(18), P5:식품첨가물(33), P6:곰팡이독소(12), P9:기타(33)
- **동작**: 라벨 클릭 → 해당 분류 항목 드롭다운 표시 + "전체 추가" 일괄추가 바
- **대분류/소분류 모드**: 여러 대분류(A,B,C...)면 대분류 표시, 단일 대분류(P)면 소분류(P1,P2...) 표시
- **필드 매칭**: 같은 이름 "참고용"이 여러 개(식품F/축산L/기타E) → 현재 분야 코드로 정확한 purpose 매칭

### UI 개선
- **가로 한 줄 정렬**: `flex-wrap: nowrap` + `overflow-x: auto` + 스크롤바 숨김
- **전체 폭 배치**: food-type-group(좁은 그리드 칼럼) 안에서 → sample-label 아래 전체 폭 행으로 이동
- 레이아웃 순서: 📌 시료 → [라벨 칩] → 제품시료명/보고번호/시험항목 검색

### 멀티셀렉트 지원
- **문제**: 항목 선택 시 드롭다운이 닫혀서 한 번에 하나만 선택 가능

- `onRefCatLabelClick(cat, refresh)` — refresh 파라미터 추가로 재호출 시 토글 방지
- 다른 라벨로 전환하여 계속 선택 가능
