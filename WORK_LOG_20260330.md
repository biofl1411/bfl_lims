## 1. 재무관리 > 입금 관리 (`paymentMgmt.html`)

### 1-1. 방안 A — 단방향 입금 연동 (접수현황 `payDate`/`payMethod` 동기화)

**문제**: 입금관리(`paymentMgmt.html`)와 접수현황(`receiptStatus.html`)이 서로 다른 필드를 사용하여 결제 상태가 연동되지 않음
- 입금관리: `receipts.payments[]` 배열에 이력 누적
- 접수현황: `receipts.payDate` 단일 필드로 결제 완료 판단

**구현**: `processPayment()` 함수에 연동 로직 추가
- 입금 처리 후 **누적 입금액 ≥ 총수수료**이면 → `payDate`, `payMethod` 필드도 함께 업데이트
- `deletePayment()` 에서 삭제 후 **미완료 상태**가 되면 → `payDate`, `payMethod` 필드 삭제(`FieldValue.delete()`)

```
입금관리에서 전액 입금 처리
  → receipts[id].payments[] 배열 추가 (기존)
  → receipts[id].payDate = 입금일자  (신규 연동)
  → receipts[id].payMethod = 입금방법 (신규 연동)
  → 접수관리>접수현황의 결제 컬럼 즉시 반영됨
```

### 1-2. 계산서 발행여부 컬럼 추가

목록 테이블에 **계산서** 컬럼 추가 (11번째 컬럼, 상태 앞)

| 표시 | 조건 | 색상 |
|---|---|---|
| 발행완료 + 발행일 | `taxInvoiceIssued=true` | 초록 |

### 2-1. 입금현황 컬럼 추가 (미입력 탭 + 전체 탭)

두 탭의 테이블에 **입금현황** 컬럼 추가

| 표시 | 조건 |
|---|---|
| 입금완료 + 날짜 | `payments[]` 누적 ≥ 총수수료 또는 `payDate` 있음 |

- `getPayStatusTagRF(d)` — 상태 태그 HTML 반환

→ **세금계산서 발행 작업 중 해당 건의 입금 여부를 입금관리 이동 없이 확인 가능**

### 2-2. 접수 상세 팝업 → 인라인 아코디언 행으로 변경

**이전**: 접수번호 클릭 시 화면 중앙 오버레이 팝업
**이후**: 해당 행 바로 아래로 펼쳐지는 아코디언 행

**동작 방식**:
- 접수번호 클릭 → 해당 `<tr>` 하이라이트(파란 배경) + 바로 아래 확장 행 삽입
- 같은 접수번호 재클릭 → 접기 (토글)
- 닫기 버튼 또는 다른 행 클릭 시 이전 확장 행 자동 닫힘

**확장 행 내용 (3단 구성)**:
1. 접수번호·접수일·담당자·업체명·사업자번호·대표자·이메일 요약
2. 시료 및 검사항목 테이블
3. 💰 결제/계산서 입력 바 (결제방법·계산서유형·과세구분·발행예정일 + 저장·닫기)

**추가 버그 수정**: 시료 테이블에서 검사항목이 `[object Object]` 로 표시되던 문제
- `s.testItems`가 객체 배열이므로 `.name` / `.itemName` 등 필드로 안전하게 추출

**JS 오류 수정**: 오버레이 HTML 제거 후 잔여 이벤트 리스너(`detailOverlay.addEventListener`)가 남아 페이지 전체 스크립트 중단되는 오류 수정

## 3. 페이지 간 연결 구조

```
receipts (Firestore)
  │
  ├── payments[]         ← paymentMgmt.html 에서 추가/삭제
  │     └── [방안A] 전액 완료 시 ↓
  ├── payDate            ← paymentMgmt.html 에서 동기화 (방안A)
  ├── payMethod          ← paymentMgmt.html 에서 동기화 (방안A)
  │     └── receiptStatus.html 에서 읽어 결제 상태 표시
  │
  ├── taxInvoiceIssued   ← receiptFinance.html / taxInvoice.html 에서 발행 처리
  ├── invoiceType        ← receiptFinance.html 미입력 탭에서 입력
  └── invoiceDate        ← receiptFinance.html 에서 입력

paymentMgmt.html (입금 관리)
  - payments[] 읽기/쓰기
  - 계산서 컬럼: taxInvoiceIssued, invoiceType, payMethod 읽기

receiptFinance.html (재무 접수현황)
  - 입금현황: payments[], payDate 읽기
  - 결제/계산서 입력: payMethod, invoiceType, taxCategory, invoiceDate 쓰기

receiptStatus.html (접수관리 접수현황)
  - 결제 상태: payDate 읽기 (방안A로 paymentMgmt.html과 연동됨)
```
